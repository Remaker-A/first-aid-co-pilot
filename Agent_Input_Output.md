# FirstAid Copilot Agent 输入输出说明

本文档用于明确 FirstAid Copilot 中 **Edge Gemma / Agent 决策引擎** 的输入、输出与协作边界，方便 Agent、视觉识别、Android 终端、语音交互和工具链同学对齐接口。

## 1. Agent 职责边界

Agent 负责把多源感知信息转化为下一步急救指导动作。

核心链路：

```text
PerceptionEvent + SessionState + FirstAidKnowledge
        ↓
Gemma Agent Decision Engine
        ↓
GuidanceAction + SessionLog + HandoverReport
```

简单来说：

- Agent 不直接负责摄像头识别。
- Agent 不直接负责 Android UI 实现。
- Agent 不直接负责拨打 120、震动、录制等系统能力。
- Agent 负责根据输入判断当前急救阶段，并输出下一步可执行指导。

## 2. 总体输入 Input

| 输入类别 | 来源 | Agent 接收内容 | 典型用途 |
| --- | --- | --- | --- |
| 用户语音反馈 | Speech to Text | 用户语音转写后的文本 | 判断用户反馈、回答用户问题、推进流程 |
| 患者视觉状态 | 后置摄像头 / 视觉模型 | 患者是否倒地、是否有反应、是否有胸廓起伏、是否抽搐 | 辅助判断是否疑似心脏骤停 |
| CPR 动作识别 | 后置摄像头 / CPR 感知识别 | 手部位置、按压频率、中断时长、手臂是否伸直、置信度 | 实时纠正 CPR 动作 |
| 施救者状态 | 前置摄像头 / 麦克风 | 恐慌、疲劳、犹豫、停顿、语速急促 | 调整语气、提醒换人、鼓励继续 |
| 场景与设备状态 | Android 终端 | GPS、摄像头权限、麦克风权限、是否离线、是否已拨打 120、是否录制中 | 决定工具调用与流程分支 |
| 会话状态 | Agent 内部状态机 | 当前急救阶段、已完成步骤、CPR 开始时间、按压次数、最近一次中断 | 保持流程连续性，避免重复指令 |
| 本地急救知识库 | 医学流程 / 指南库 | CPR 步骤、成人/儿童/婴儿差异、禁忌提示、安全边界 | 约束 Agent 输出，避免危险建议 |

## 3. 总体输出 Output

| 输出类别 | 输出对象 | Agent 输出内容 | 下游模块 |
| --- | --- | --- | --- |
| 语音引导 | `TTSInstruction` | 简短、明确、一步一步的急救指令 | TTS / Android |
| 屏幕提示 | `UIInstruction` | 大字提示、状态标签、质量分、倒计时、下一步动作 | Android UI |
| 视觉辅助提示 | `VisualOverlayInstruction` | 按压区域高亮、手部位置提示、动画提示 | Android UI / 视觉辅助层 |
| 触觉节拍 | `HapticAction` | 100-120 次/分钟震动节拍 | Android 震动模块 |
| 工具调用 | `ToolAction` | 拨打 120、发送 GPS、开始本地录制、查找 AED、生成报告 | Android / 工具链 |
| 实时纠错 | `CorrectionAction` | 位置偏左、频率过慢、手臂没伸直、按压中断等纠正建议 | TTS + UI |
| 澄清问题 | `ClarifyingQuestion` | 向施救者确认患者是否有反应、是否有正常呼吸等 | 用户语音交互 |
| 会话日志 | `SessionLog` | 每一步判断、用户反馈、CPR 质量数据、干预事件 | 报告生成 / 回放 |
| 交接报告 | `HandoverReport` | 救护车到达时给医护人员的结构化摘要 | NFC / 本地展示 / 分享 |

## 4. 阶段一：引导阶段

引导阶段的目标是让普通用户在紧急状态下快速完成初步判断，并在必要时启动 CPR。

### 4.1 输入 Input

| 输入 | 示例 | 说明 |
| --- | --- | --- |
| 启动事件 | 用户点击“一键急救” | 表示进入急救流程 |
| 用户语音反馈 | “他没有反应”“我拍了肩膀，没有动”“他好像还有呼吸” | 由 STT 模型转为文本后传给 Agent |
| 患者视觉初筛结果 | 倒地、闭眼、无明显肢体反应、胸廓无明显起伏 | 由视觉模型输出结构化结果 |
| 麦克风环境反馈 | 是否听到呻吟、咳嗽、回应 | 用于辅助判断患者是否有反应 |
| 设备状态 | 摄像头可用、麦克风可用、GPS 可用、离线/在线 | 决定是否能进入完整流程 |

### 4.2 输出 Output

| 输出 | 示例 | 说明 |
| --- | --- | --- |
| 主动语音引导 | “请靠近患者，大声呼叫他，并轻拍他的肩膀。” | 一步一步指导用户判断意识 |
| 判断问题 | “他有回应、咳嗽或明显动作吗？” | 让用户补充关键判断信息 |
| 下一步指令 | “请观察胸口是否有正常起伏。” | 引导用户判断呼吸 |
| 流程分支决策 | 有反应则进入观察/呼叫帮助；无反应且无正常呼吸则进入 CPR | Agent 状态机推进 |
| 工具调用请求 | 拨打 120、发送 GPS、开始本地录制 | 输出给 Android / 工具链执行 |
| 安全提示 | “如果不确定是否正常呼吸，请按无正常呼吸处理，并立即呼叫急救。” | 降低犹豫成本 |

## 5. 阶段二：视觉辅助阶段

视觉辅助阶段的目标是在 CPR 执行过程中结合视觉识别结果，对施救者进行实时纠错和节奏引导。

### 5.1 输入 Input

| 输入 | 示例 | 说明 |
| --- | --- | --- |
| 患者状态视觉结果 | 无明显胸廓起伏、疑似无自主呼吸、疑似恢复呼吸 | 决定是否继续 CPR 或提示复查 |
| 按压位置识别 | 手在胸口中央、偏左、偏右、过高、过低 | 用于位置纠错 |
| 按压节奏识别 | 92/min、115/min、135/min | 判断是否低于或高于 100-120 次/分钟 |
| 按压中断识别 | 中断 2 秒、中断 5 秒 | 用于提醒继续按压 |
| 手型 / 手臂识别 | 双手未交叠、手臂弯曲、手指未上翘 | 用于动作质量纠正 |
| 用户询问 | “我应该按哪里？”“我这样对吗？”“要按多快？” | Agent 结合视觉信息回答 |
| 施救者状态识别 | 恐慌、疲劳、犹豫、动作变慢 | 决定语气、安抚和换人提醒 |

### 5.2 输出 Output

| 输出 | 示例 | 说明 |
| --- | --- | --- |
| 按压位置指导 | “把双手掌根放到胸口中央，两乳头连线的中点。” | 回答用户位置问题 |
| 实时位置纠正 | “位置稍微向右一点。”“手再往下移一点。” | 根据视觉模型结果纠正 |
| 节奏引导 | “跟着震动按，保持每分钟 100 到 120 次。” | 结合触觉节拍器 |
| 频率纠正 | “再快一点。”“稍微慢一点，跟着我的节奏。” | 根据按压频率输出 |
| 连续性提醒 | “不要停，继续按压。” | 中断超过阈值时触发 |
| 姿势纠正 | “手臂伸直，用上半身向下压。” | 根据手臂/手型识别结果触发 |
| 情绪安抚 | “你做得很好，我会一步一步陪你。” | 检测到恐慌时触发 |
| 换人提醒 | “如果旁边有人，请准备换手。” | 检测到疲劳或持续按压较久时触发 |
| 视觉 UI 指令 | 高亮按压区域、显示质量分、显示节拍器 | 输出给 Android UI |

## 6. 推荐统一接口对象

为了减少各模块之间的沟通成本，建议统一围绕三个对象协作：

```text
PerceptionEvent  ->  GuidanceAction  ->  SessionLog / HandoverReport
```

### 6.1 PerceptionEvent

`PerceptionEvent` 是传给 Agent 的统一输入事件，包含语音、视觉、设备和流程状态。

示例：

```json
{
  "event_id": "evt_001",
  "timestamp": "2026-05-28T18:10:23+08:00",
  "stage": "cpr_feedback",
  "stt_text": "我这样按对吗",
  "patient_state": {
    "responsive": false,
    "normal_breathing": false,
    "confidence": 0.86
  },
  "cpr_quality": {
    "hand_position": "left_offset",
    "compression_rate": 92,
    "interruption_seconds": 0,
    "arm_straight": false,
    "confidence": 0.81
  },
  "rescuer_state": {
    "emotion": "anxious",
    "fatigue_level": "medium"
  },
  "device_state": {
    "gps_available": true,
    "emergency_call_started": true,
    "recording": true,
    "network": "offline"
  }
}
```

### 6.2 GuidanceAction

`GuidanceAction` 是 Agent 输出给下游模块的统一动作描述。

示例：

```json
{
  "action_id": "act_001",
  "timestamp": "2026-05-28T18:10:24+08:00",
  "stage": "cpr_feedback",
  "priority": "high",
  "tts": {
    "text": "位置稍微向右一点，手臂伸直，跟着震动继续按压。",
    "tone": "calm",
    "speed": "normal"
  },
  "ui": {
    "main_text": "按压位置偏左",
    "secondary_text": "向右调整，继续按压",
    "quality_score": 68
  },
  "visual_overlay": {
    "highlight_target": "chest_center",
    "correction_arrow": "right"
  },
  "haptic": {
    "enabled": true,
    "metronome_bpm": 110
  },
  "tool_action": null,
  "log_event": {
    "type": "correction",
    "detail": "hand_position_left_offset"
  }
}
```

### 6.3 SessionLog

`SessionLog` 用于记录完整急救过程，后续生成交接报告。

建议记录内容：

| 字段 | 示例 |
| --- | --- |
| 初判时间 | 14:32:07 |
| 症状判断 | 意识丧失、无正常呼吸、疑似心脏骤停 |
| CPR 开始时间 | 14:32:29 |
| 累计按压次数 | 487 次 |
| 平均按压频率 | 112/min |
| 平均质量评分 | 87/100 |
| 中断事件 | 14:33:15 中断 3 秒 |
| 干预事件 | 呕吐物处理、换人、疑似恢复呼吸 |
| AED 状态 | 未找到 / 已使用 |
| 视频记录 | 已本地保存 |

### 6.4 HandoverReport

`HandoverReport` 是给急救员或医院的结构化交接摘要。

示例：

```text
患者ID：匿名
初判时间：14:32:07
症状：意识丧失，无正常呼吸，疑似心脏骤停
CPR 开始：14:32:29
累计按压：487 次，持续 4 分 38 秒
平均频率：112/min
质量评分：87/100
干预事件：
- 14:33:15 呕吐物处理
- 14:35:02 第一次换手
- 14:36:45 疑似自主呼吸出现 5 秒，后消失
AED 使用：否，现场未找到
施救者状态：疲劳，已换人 1 次
视频记录：已保存本地，可分享给医院
```

## 7. Agent 与其他模块协作关系

| 模块 | 向 Agent 提供 | 接收 Agent 输出 |
| --- | --- | --- |
| 语音 / STT | 用户语音转写文本 | 需要 TTS 播报的文本 |
| 视觉识别 | 患者状态、手部位置、按压质量、施救者状态 | 需要高亮或纠正的目标 |
| Android 终端 | 设备权限、GPS、录制状态、拨号状态 | UI 文案、震动节拍、系统工具调用 |
| 工具链 | AED 信息、120 状态、报告生成能力 | 拨号、定位、录制、报告等调用请求 |
| 医学 / 产品 | CPR 流程、禁忌提示、免责声明 | 需要审核的提示语和决策边界 |

## 8. MVP 阶段建议优先级

第一优先级：

- 启动后主动引导用户判断意识和呼吸。
- 根据用户语音反馈推进流程。
- 判断是否进入 CPR。
- 输出 CPR 位置、节奏和连续性指导。
- 记录 SessionLog。

第二优先级：

- 结合视觉识别结果纠正按压位置。
- 结合按压频率输出“快一点 / 慢一点”。
- 中断超过阈值时提醒继续。
- 生成基础 HandoverReport。

第三优先级：

- 识别施救者恐慌和疲劳。
- 自动调整语速和语气。
- 多人换手提醒。
- AED 协作与多人调度。

## 9. 一句话总结

Agent 的核心职责是：

> 接收用户语音反馈、视觉感知结果和设备状态，结合本地急救知识库与当前会话状态，输出下一步可执行的语音引导、视觉纠错、节拍提示、工具调用和交接日志。
